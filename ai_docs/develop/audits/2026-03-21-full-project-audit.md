## Высокий приоритет

### [A1] Монолитный `js/i18n.js` («контент + рантайм»)

**Категория**: архитектура  
**Суть**: все строки RU/EN в одном объекте рядом с логикой DOM; полный словарь на каждой странице; нет автоматической связи `data-i18n*` с ключами.  
**Исправление**: по мере роста — разнести словари (JSON/чанки по страницам), опционально ленивая загрузка; проверки покрытия ключей (скрипт/CI).

### [A2] Хрупкий контракт `scripts/refresh-layout.mjs`

**Категория**: архитектура  
**Суть**: вставка шапки/подвала через маркеры и обход вложенности `</div>`; при сдвиге разметки страницы могут тихо пропускаться; `NAV_ACTIVE_BY_FILE` дублирует знание о маршрутах.  
**Исправление**: более устойчивый парсинг или шаблонизатор; единый источник правды для активного пункта меню; явные ошибки/exit code при skip.

### [A3] Нет устойчивой границы «сайт JS» / «страница JS»

**Категория**: архитектура  
**Суть**: общее поведение в `js/script.js`, но лайтбокс и `slidesData` в `licenzii.html`, форма в `kontakty.html`, MapLibre в `opyt.html` — inline.  
**Исправление**: вынести страничную логику во внешние модули/файлы с предсказуемым порядком загрузки.

### [S1] Сторонние ресурсы без SRI и со слабым pinning

**Категория**: безопасность  
**Место**: Font Awesome (cdnjs), Google Fonts (`@import` в CSS), MapLibre с unpkg (`maplibre-gl@4`), стиль карт с внешних хостов.  
**Влияние**: риск supply-chain при компрометации CDN/реестра или DNS.  
**Исправление**: `integrity` + `crossorigin` для фиксированных версий; self-host где уместно; политика на сервере (CSP).

## Средний приоритет

**Архитектура**

- **[A4]** Монолитный `css/style.css` (~950 строк) — сложно сопровождать и безопасно рефакторить без разбиения или сборки.  
- **[A5]** `js/script.js` на каждой странице — растёт parse/download cost при добавлении поведения.  
- **[A6]** Лайтбокс лицензий: глобальные функции и `onclick` в разрез со стилем IIFE в остальном коде.  
- **[A7]** Несогласованная доставка third-party (cdnjs / Google Fonts / unpkg) без единой политики pinning/SRI/fallback.  
- **[A8]** `data-i18n-html` → `innerHTML` в `applyTranslations` — архитектурный footgun при появлении недоверенных строк.  
- **[A9]** Несколько механизмов билингва (дефолты в HTML, `i18n.js`, отдельный `slidesData` в `licenzii.html`).

**Безопасность**

- **[S2]** Тот же `innerHTML` для переводов — при CMS/внешних строках нужна санитизация или отказ от HTML в строках.  
- **[S3]** Нет CSP и смежных заголовков; строгий CSP осложнён inline-скриптами — перенос в файлы или nonces/hashes.  
- **[S4]** Форма на `kontakty.html`: UI «отправлено» без реальной отправки — разрыв ожиданий; при появлении API — валидация и честные тексты.

**Качество кода**

- **[Q1]** Табы: `document.querySelector('#' + hash)` может бросить исключение и оборвать инициализацию слушателей.  
- **[Q2]** i18n обновляет `document.title`, но не `<meta name="description">` и прочий head.  
- **[Q3]** Тексты лицензий в `slidesData` в `licenzii.html` дублируют смысл с `i18n.js` — риск рассинхрона.  
- **[Q4]** Scroll-reveal завязан на фиксированный список классов — новые блоки без правки списка не анимируются.  
- **[Q5]** Карта в `opyt.html`: нет обработки ошибок загрузки стиля/тайлов и запасного UI.  
- **[Q6]** `refresh-layout.mjs`: синхронное I/O без `try/catch` — падение без структурированного отчёта.  
- **[Q7]** Пустые `catch` у `localStorage` в i18n — молчаливый сбой языка.  
- **[Q8]** Много inline `style` на страницах — дублирование с `style.css`, усложнение редизайна.

## Низкий приоритет / пожелания
