## Критические проблемы (исправить немедленно)

### [A1] / [Q1] Двойная загрузка `script.js` и `i18n.js` на `licenzii.html`

**Категория**: архитектура и качество кода  
**Место**: `licenzii.html` — скрипты подключены дважды (в середине страницы и после подвала).  
**Влияние**: повторная инициализация i18n, повторная привязка слушателей к `.lang-btn`, гамбургеру и др.; лишняя нагрузка, риск «двойных» срабатываний и затруднённая отладка.  
**Исправление**: оставить **одну** пару тегов `<script src="js/script.js">` и `<script src="js/i18n.js">` в конце документа (или в согласованном с остальными страницами месте), удалить дубликат.

## Высокий приоритет (исправить в ближайшее время)

### [A2] Дублирование шапки, мобильного меню и подвала

**Место**: ~11 HTML-файлов с полными копиями блоков.  
**Влияние**: любое изменение навигации или контактов требует правок во многих файлах; высокий риск рассинхрона.  
**Исправление**: сборка из шаблонов (Eleventy, Vite plugin, простые includes), или серверные includes, или единый JS-инжект (менее предпочтительно для SEO/без JS).

**Статус: исправлено** (2026-03-21). Единый источник: `partials/header.html`, `partials/footer.html`; развёртывание во все корневые `*.html` — `node scripts/refresh-layout.mjs` (`scripts/refresh-layout.mjs`). Активный пункт меню: атрибут `data-nav-active` на `<body>` (карта в скрипте) + `data-nav-match` в шапке и логика в `js/script.js`. **Замечание по корректности:** в закоммиченных HTML по-прежнему лежит развёрнутая копия блоков (ожидаемо для статической выдачи без SSI); расхождения устраняются повторным запуском скрипта после правок partials, а не правкой 11 файлов вручную.

### [A3] Монолитный `js/i18n.js`

**Место**: `js/i18n.js` (~1200 строк), пары `ru` / `en` в одном файле.  
**Влияние**: каждая страница тянет полный словарь; конфликты при слиянии веток; сложнее масштабировать локали и разбивку по разделам.  
**Исправление**: разнести по JSON и подгружать по странице/локали, или модульная сборка — по мере роста проекта.

**Статус: исправлено** (2026-03-21). Словари вынесены в `i18n/locales/`: общий пакет `common.{ru,en}.json` (навигация, подвал, кнопки, хлебные крошки и т.д.) и отдельные пары по страницам (`index`, `sorbenty`, `energosistemy`, …). `js/i18n.js` — компактный загрузчик: по `data-page-title` на `<body>` подтягиваются только `common` + нужная страница; переключение языка подгружает вторую локаль с кэшем в памяти. **Ограничение:** подстановка через `fetch` — просмотр с протоколом `file://` без локального HTTP-сервера может не подгрузить JSON; для разработки и продакшена предполагается выдача по `http(s)://`. Регенерация из старого монолита (если восстановить бэкап): `node scripts/migrate-i18n-to-json.mjs`.

## Средний приоритет (запланировать)

**Архитектура**

- **[A4]** HTML внутри строк переводов (`data-i18n-html`, inline-стили в текстах) — связка контента, вёрстки и i18n; усложняет вынос в CMS и санитизацию.  
- **[A5]** Часть стилей в inline `<style>` на отдельных страницах при общем `css/style.css` — фрагментирование стилей.  
- **[A6]** У многих страниц нет `<meta name="description">` (в отличие от главной) — слабее SEO и сниппеты.

**Безопасность**

- **[S1]** Скрипты/стили с CDN без SRI; нет CSP в разметке — усиление ударного эффекта при компрометации CDN (нужны заголовки/SRI на хостинге).  
- **[S2]** `innerHTML` в `applyTranslations` для `data-i18n-html` — безопасно пока строки только из репозитория; при удалённых переводах нужна санитизация.  
- **[S3]** Форма на `kontakty.html` — клиентский заглушечный успех без отправки; при появлении бэкенда — валидация, CSRF, честные ожидания пользователя.  
- **[S4]** Дублирование скриптов на `licenzii.html` — побочные эффекты для клиентской логики (см. A1).

**Качество кода**

- **[Q2]** Нет `package.json`, тестов, линтера — регрессии только ручной проверкой.  
- **[Q3]** Табы по hash в `js/script.js`: `document.querySelector('#' + hash)` может быть небезопасен для некоторых значений hash — стоит `CSS.escape` или `getElementById` с проверкой.  
- **[Q4]** Глобальные inline-функции на отдельных страницах (`handleForm`, лайтбокс) vs IIFE в `script.js` — размытая зона ответственности.  
- **[Q5]** Пропущенные ключи i18n не подсвечиваются — остаётся русский fallback без сигнала в dev.

## Низкий приоритет / пожелания

**Архитектура:** [A7] заглушка формы без бэкенда; [A8] внешние CDN и картинки; [A9] длинный список селекторов для scroll-reveal в `script.js`.  

**Безопасность:** [S5] ссылка `http://atomsmk.ru` — предпочтительно HTTPS; [S6]–[S9] localStorage языка, mailto/tel, `rel="noopener"`, сторонние iframe карт — ожидаемо для маркетингового сайта.  

**Качество:** [Q6] магические числа в `script.js`; [Q7] счётчик статистики без защиты от `NaN`; [Q8] смешение `var`/`const` в старом монолитном `i18n.js` — загрузчик компактный, словари в JSON (**A3**); при желании унифицировать стиль в малом `i18n.js`.

## Remediation Applied

**Дата**: 2026-03-21  

### Исправлено

- **[A1] / [Q1]** На `licenzii.html` удалена первая пара тегов `<script src="js/script.js">` и `<script src="js/i18n.js">` (перед инлайн-скриптом лайтбокса). Глобальные скрипты подключаются один раз в конце `</body>`, после футера — как на остальных страницах.

- **[A2]** Введены общие фрагменты `partials/header.html` (шапка + `.mobile-nav`) и `partials/footer.html`. Скрипт `scripts/refresh-layout.mjs` заменяет соответствующие блоки во всех 11 HTML в корне проекта и проставляет `data-nav-active` на `<body>` по имени файла. В `js/script.js` добавлена инициализация класса `active` у элементов с `data-nav-match`. Проверка: на каждой странице ровно один `<header class="header">`; источник правок для навигации и подвала — только partials + повторный запуск скрипта.

- **[A3]** Монолитные строки вынесены в `i18n/locales/*.json` (пакет `common` + один пакет на раздел сайта). `js/i18n.js` подгружает по `fetch` только `common` и пакет текущей страницы (сопоставление `data-page-title` → имя пакета в `TITLE_KEY_TO_BUNDLE`); переключатель RU/EN кэширует загруженные словари. Миграция из прежнего одного файла: `node scripts/migrate-i18n-to-json.mjs` (однократно, пока в `i18n.js` ещё есть литерал `const translations = { ... }`).
