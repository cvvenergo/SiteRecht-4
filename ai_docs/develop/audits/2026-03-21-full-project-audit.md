## Высокий приоритет (исправить в ближайшее время)

### [A2] Дублирование шапки, мобильного меню и подвала — **устранено**

**Категория**: архитектура  
**Исправление** (2026-03-21): единые фрагменты `partials/header.html` и `partials/footer.html`; скрипт `scripts/refresh-layout.mjs` заменяет блок от `<!-- HEADER -->` / `<header class="header">` до конца `.mobile-nav` и блок футера во всех 11 корневых `*.html`. На `<body>` задаётся `data-nav-active` (home | directions | management | experience | licenses | contacts); в `js/script.js` класс `active` на десктопных `.nav-link[data-nav]` выставляется по этому атрибуту. После правок шапки/подвала: **`node scripts/refresh-layout.mjs`** из корня проекта (нужен Node.js).

### [S1] Сторонние скрипты и стили без SRI

**Категория**: безопасность  
**Место**: подключения с CDN (например Font Awesome, MapLibre с unpkg, шрифты через `@import` в CSS).  
**Влияние**: при компрометации CDN возможна подмена исполняемого кода в origin сайта.  
**Исправление**: Subresource Integrity для известных версий; по возможности self-host; политика на стороне сервера (CSP).

### [S2] Форма на `kontakty.html` без отправки на сервер

**Категория**: безопасность / доверие  
**Место**: клиентский обработчик формы (например `handleForm`, `preventDefault`, показ «успеха»).  
**Влияние**: пользователь может считать, что ПДн отправлены; фактической передачи и хранения нет — разрыв ожиданий и соответствия законодательству при реальном сборе данных.  
**Исправление**: при появлении бэкенда — POST по HTTPS, валидация, честные сообщения; до этого — явный текст, что это демо/заглушка.

### [Q1] Дублирование полного каркаса страницы — **устранено**

**Категория**: качество кода  
См. **[A2]** — тот же источник правды и процесс поддержки.

### [Q2] Повторная загрузка `script.js` на `licenzii.html` (поведение) — **устранено**

**Категория**: качество кода  
Следствие [A1]; закрыто вместе с удалением дубликата скриптов.

### [Q3] Монолитный `js/i18n.js`

**Категория**: качество кода  
**Место**: один большой объект `translations` в `js/i18n.js`.  
**Влияние**: конфликты при слиянии веток, рост payload на каждой странице, сложность навигации; опечатки в `data-i18n` проявляются поздно.  
**Исправление**: разнесение по JSON/модулям и ленивая загрузка по странице по мере роста проекта.

## Средний приоритет (запланировать)

**Архитектура**

- **[A3]** Весь RU/EN в одном файле — рост размера и конфликтов при масштабировании контента.  
- **[A4]** Смешение inline `<style>` / `<script>` на отдельных страницах с общими `style.css` и `script.js` — нечёткие границы слоёв.  
- **[A5]** `innerHTML` для ключей `data-i18n-html` — при появлении строк из CMS/внешних источников нужна санитизация и политика.  
- **[A6]** Широкие селекторы в `script.js` — жёсткая связка разметки и поведения.

**Безопасность**

- **[S3]** Нет Content-Security-Policy и смежных заголовков — нет defense-in-depth при будущих уязвимостях.  
- **[S4]** То же использование `innerHTML` в `applyTranslations` — классический XSS-sink при неверном источнике строк.  
- **[S5]** Встраивание карт/iframe и сторонних API — расширение поверхности и зависимость от третьих сторон.

**Качество кода**

- **[Q4]** HTML и inline-стили внутри строк переводов — связка копирайта с вёрсткой.  
- **[Q5]** Нет автотестов и линтеров для HTML/JS.  
- **[Q6]** Scroll-reveal через инлайн-стили в JS — хрупче, чем классы в CSS.  
- **[Q7]** Несогласованные префиксы ключей i18n — выше риск ошибок при правках.  
- **[Q8]** Размытая зона между inline-скриптами страниц и `script.js`.

## Низкий приоритет / пожелания

**Архитектура:** [A7] зависимость от CDN и внешних изображений; [A8] нет видимого CI (валидация HTML, проверка ссылок).

**Безопасность:** [S6] инлайн-обработчики (`onclick`, `onsubmit`) усложняют строгий CSP; ~~[S7] дублирование скриптов на `licenzii.html`~~ — снято вместе с **A1**; [S8] ссылки на `http://` при ожидании HTTPS; [S9] секретов в репозитории не обнаружено — поддерживать так же.

**Качество:** [Q9] магические числа в `script.js`; [Q10] пустые `catch` при работе с `localStorage` в i18n; [Q11] неиспользуемые ключи переводов без проверки; [Q12] счётчики без проверки `NaN` для `data-count`.
