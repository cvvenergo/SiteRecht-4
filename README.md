# Проект 2 — сайт РЭХТ

Статический многостраничный сайт (HTML/CSS/JS). Общая шапка и подвал лежат в `partials/` и подмешиваются в корневые `*.html` скриптом развёртывания.

## Требования

- **Node.js** 18+ (для ESM и `import` в скрипте развёртывания). Отдельного `npm install` в корне нет — runtime-зависимостей через npm сайт не тянет.

## Обновление вёрстки (шапка / подвал / активный пункт меню)

Из **корня репозитория**:

```bash
node scripts/refresh-layout.mjs
```

Скрипт читает `partials/header.html` и `partials/footer.html`, вставляет их в разметку страниц и выставляет на `<body>` атрибут `data-nav-active` по имени файла. Соответствие «файл → ключ навигации» задаётся в **`NAV_ACTIVE_BY_FILE`** внутри `scripts/refresh-layout.mjs` (при добавлении новой корневой страницы обновите и карту, и сами partials при необходимости).

### Контракт partials

- В `partials/header.html` первый узел — комментарий `<!-- HEADER -->` и корневой `<header class="header">` (как ожидает скрипт).
- В `partials/footer.html` — `<!-- FOOTER -->` и `<footer class="footer">`.
- В сгенерированных страницах навигация подсвечивается через `data-nav-active` и `data-nav` на ссылках (логика в `js/script.js`).

После правок **только** в `partials/*` обязательно снова запустите `refresh-layout.mjs`, иначе корневые HTML останутся со старой вставкой.

## Сторонние ресурсы: pinning и SRI

- **Font Awesome** (cdnjs): фиксированная версия в URL, подключение с **`integrity`** и **`crossorigin="anonymous"`** на страницах.
- **MapLibre GL** (`opyt.html`): фиксированная patch-версия (например `4.7.1`), CSS и JS с **`integrity`** и **`crossorigin="anonymous"`**.
- **Google Fonts**: стили подключаются обычным **`<link href="https://fonts.googleapis.com/...">`** в `<head>`. Для ответа Google отдаёт CSS, зависящий от User-Agent, поэтому **SRI на этот CSS не используется** (см. комментарии в HTML рядом с тегами). Шрифты грузятся с `fonts.gstatic.com` с `preconnect`.
- Тайлы и стиль карты MapLibre по-прежнему с внешних URL — для них действуют ограничения CDN/сети, не полный SRI-контроль как у одного файла библиотеки.

## Деплой

Проект рассчитан на **статический хостинг**: выкладывайте корень репозитория (или собранный снимок с уже прогнанным `refresh-layout.mjs`) на любой хост без серверного рантайма. Достаточно отдачи файлов по HTTPS; отдельной сборки фронтенда нет.
